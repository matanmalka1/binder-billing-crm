# Database and Models

## Engine Configuration

`app/database.py`:
- SQLAlchemy 2.0 engine with `pool_pre_ping=True` (validates connections before checkout).
- `echo=False` always (SQL logging goes through `StructuredFormatter` via the `sqlalchemy.engine` logger, controlled by `LOG_SQL` setting).
- Production blocks SQLite (`raise RuntimeError` if `DATABASE_URL` starts with `sqlite` in production).
- Two SQLAlchemy event hooks record query start/end times into `RequestLogStats` for per-request SQL telemetry.

## ORM Style

All queries use SQLAlchemy 2.0 `select()` style. The 1.x `Query` API (`db.query(Model)`) is not used.

```python
# Correct
stmt = select(Binder).where(Binder.id == binder_id)
result = db.scalars(stmt).first()

# Also correct for projections
stmt = select(Binder.id, LegalEntity.official_name.label("client_name"))
rows = db.execute(stmt).all()

# Not used
db.query(Binder).filter_by(id=binder_id).first()
```

## Base Class

```python
class Base(DeclarativeBase):
    pass
```

All ORM models inherit from `Base` imported from `app.database`. `model_registry.py` imports every model module before the app starts so SQLAlchemy's mapper configuration runs with all models registered.

## Model Conventions

From `app/binders/models/binder.py` as a representative example:

**Primary key**: `id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)`

**Foreign keys**: `client_record_id: Mapped[int] = mapped_column(ForeignKey("client_records.id"), nullable=False, index=True)`

**Enums**: Declared inline using Python `str, PyEnum` and wrapped with `pg_enum()` from `app/utils/enum_utils.py` for PostgreSQL-native enum types.

**Nullable columns**: `period_start: Mapped[date | None] = mapped_column(nullable=True)`

**Timestamps**: `created_at: Mapped[datetime] = mapped_column(default=utcnow, nullable=False)` — `utcnow` is from `app/utils/time_utils.py`, returns `datetime.now(UTC)`.

**Soft delete**: `deleted_at: Mapped[datetime | None] = mapped_column(nullable=True)`, `deleted_by: Mapped[int | None] = mapped_column(ForeignKey("users.id"), nullable=True)`

**Relationships**: Declared with `Mapped[list["Model"]]` and `relationship(...)`.

**Indexes**: Declared in `__table_args__` as `Index(...)`. Partial indexes use `postgresql_where=text(...)` and `sqlite_where=text(...)` from `sqlalchemy import text`.

**Unique constraints**: Also in `__table_args__`, using `Index(..., unique=True, postgresql_where=...)` for partial uniqueness (e.g. unique binder number per active client).

## Soft Delete Pattern

Models with `deleted_at` are soft-deleted. Queries must include `.where(Model.deleted_at.is_(None))` to exclude deleted rows. `BaseRepository.soft_delete()` sets `deleted_at = utcnow()` and optionally `deleted_by`.

There is no automatic filter — repositories must explicitly exclude soft-deleted rows in every query where that matters.

## Enum Handling

Python enums used as column types:

```python
from enum import Enum as PyEnum
from app.utils.enum_utils import pg_enum

class BinderLocationStatus(str, PyEnum):
    IN_OFFICE = "in_office"
    READY_FOR_HANDOVER = "ready_for_handover"
    HANDED_OVER = "handed_over"

location_status: Mapped[BinderLocationStatus] = mapped_column(
    pg_enum(BinderLocationStatus),
    default=BinderLocationStatus.IN_OFFICE,
    nullable=False,
)
```

`str, PyEnum` means the enum value is a string, which serializes cleanly in Pydantic schemas.

## Migrations

Canonical database and migration rules: see `../../../docs/docs/architecture/database.md` and `../../../docs/docs/architecture/migrations.md`. The commands below are backend-local detail.

All schema changes go through Alembic. Never modify the DB schema manually.

```bash
# Generate migration after model changes
APP_ENV=development ENV_FILE=.env.development ./.venv/bin/python -m alembic revision --autogenerate -m "<description>"

# Apply
APP_ENV=development ENV_FILE=.env.development ./.venv/bin/python -m alembic upgrade head
```

Migration files: `alembic/versions/NNNN_<description>.py` with a sequential numeric prefix. `down_revision` must point to the previous migration. Review every autogenerated partial index — replace raw `sqlite_where`/`postgresql_where` strings with `sa.text(...)`.

Production startup runs `alembic upgrade head` before gunicorn.

## ApiDateTime and ApiDecimal

Standard response schemas use `ApiDateTime` and `ApiDecimal` from `app/core/api_types.py` instead of raw `datetime` and `Decimal` where practical:

- `ApiDateTime`: normalizes naive datetimes to UTC, serializes as `"2026-01-02T03:04:05Z"`.
- `ApiDecimal`: serializes as a string with 2 decimal places (`"123.45"`), avoids float precision issues.

New response fields such as `created_at`, `updated_at`, `handed_over_at` etc. should use `ApiDateTime`; a few legacy schemas still expose raw `datetime`/`Decimal` and should be migrated when touched.
